# Accessibility Audit & WCAG 2.1 AA Compliance Report

**Date**: March 17, 2026  
**App**: FaithLight  
**Target Standard**: WCAG 2.1 Level AA

---

## Executive Summary
The app has a **good foundation** for accessibility but needs improvements in:
- Keyboard navigation flow and skip links
- Semantic HTML structure
- Color contrast on interactive elements
- Focus indicators visibility
- Form accessibility patterns

---

## Critical Issues (Must Fix)

### 1. **No Skip-to-Content Link**
- **WCAG**: 2.4.1 Bypass Blocks (Level A)
- **Impact**: Screen reader users must navigate through header/nav before reaching main content
- **Fix**: Add skip link to all pages
- **Priority**: CRITICAL

### 2. **Interactive Elements Lack Proper Focus Indicators**
- **WCAG**: 2.4.7 Focus Visible (Level AA)
- **Impact**: Keyboard users can't see where focus is
- **Current**: Some elements use `focus-visible:ring` but not consistently
- **Fix**: Apply consistent focus styles across all interactive elements
- **Priority**: CRITICAL

### 3. **Buttons in Home Page Not Keyboard Accessible**
- **WCAG**: 2.1.1 Keyboard (Level A)
- **Issues**:
  - Line 158: `<button>` with `onClick` but no keyboard context
  - Line 204: Prayer button inside `<Link>` (nested interactive elements)
- **Fix**: Remove nested Link/button structures, use semantic buttons
- **Priority**: HIGH

### 4. **Color Contrast Issues**
- **WCAG**: 1.4.3 Contrast (Minimum) (Level AA)
- **Issues**:
  - Gray text on light backgrounds may fail at smaller sizes
  - Test required on: gray-500 on gray-50, gray-600 on blue-50
- **Fix**: Ensure 4.5:1 for text, 3:1 for UI components
- **Priority**: HIGH

---

## Major Issues

### 5. **Semantic Structure**
- **Current**: Uses `<div>` for navigation instead of `<nav>`
- **BottomTabs**: Uses `role="tablist"` but not true tabs (should be tab panel with tab buttons)
- **Fix**: Use semantic `<nav>`, proper heading hierarchy
- **Priority**: MEDIUM

### 6. **Image Alt Text Missing**
- **WCAG**: 1.1.1 Non-text Content (Level A)
- **Issues**:
  - Emoji icons (📖, 🙏) have no alt text
  - Background images lack description
- **Fix**: Wrap icons with `<span aria-label="...">` when no text label
- **Priority**: MEDIUM

### 7. **Missing Form Labels**
- **WCAG**: 1.3.1 Info and Relationships (Level A)
- **Issues**: Search inputs, prayer entry fields lack associated `<label>` elements
- **Priority**: MEDIUM

---

## Minor Issues

### 8. **Touch Target Size**
- **WCAG**: 2.5.5 Target Size (Level AAA)
- **Current**: Many buttons are 32-40px (minimum is 44x44px recommended)
- **Fix**: Ensure interactive elements are at least 44x44px
- **Priority**: LOW (AAA, optional)

### 9. **Error Messages Not Announced**
- **WCAG**: 3.3.1 Error Identification (Level A)
- **Current**: Retry button on verse load failure
- **Fix**: Add `role="alert"` to error messages, announce to screen readers
- **Priority**: MEDIUM

### 10. **Heading Hierarchy Issues**
- **WCAG**: 1.3.1 Info and Relationships (Level A)
- **Current**: 
  - Home page: `<h1>` immediately after `<p>` (date string)
  - Section titles are not proper headings
- **Fix**: Use proper `<h2>`, `<h3>` hierarchy
- **Priority**: LOW

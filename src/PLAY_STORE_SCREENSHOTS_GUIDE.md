# 📸 FaithLight Play Store Screenshots Guide

## Overview
**Goal:** 6 high-quality screenshots that tell the story in 5 seconds and drive installs.

---

## General Requirements

### Technical Specs
| Aspect | Value |
|--------|-------|
| **Orientation** | Portrait (9:16) |
| **Minimum Size** | 1440×2560px |
| **Format** | PNG or JPG |
| **Count** | 2-8 (recommend 5-6) |
| **Text** | Large, readable at small screen |

### Design Rules
- ✅ Big, bold text (24pt+)
- ✅ Clean backgrounds (no clutter)
- ✅ Highlight 1 feature per screenshot
- ✅ Use real app UI (not mockups)
- ✅ Show user benefit, not features
- ✅ Consistent branding/colors
- ✅ Include CTA on first image

## Design Asset Production

### Tool Recommendations
- **Figma**: Design & export quickly
- **Adobe XD**: Professional layouts
- **Photoshop**: Fine-tuning screenshots
- **Canva**: Templates (fast, less polished)

### Workflow
```
1. Take real app screenshots (pixel-perfect)
2. Open in Figma/Photoshop
3. Add text overlays with semi-transparent backgrounds
4. Add subtle graphics/arrows if needed
5. Export at 1440×2560px
6. Compress (under 5MB per image)
```

---

## Text Overlay Template (Figma)

```
Background (Optional):
- Rectangle, full bleed
- Gradient: Indigo (#6c5ce7) at top → 50% transparency
- Blur: 10px

Text Layers:
- Main title: 36pt bold, white, centered top half
- Sub text: 18pt regular, white, centered
- Detail: 14pt regular, light gray, bottom third

Safe areas:
- Top: 120px padding
- Bottom: 240px padding (for app bar)
- Sides: 60px padding
```

## Quality Checklist

- [ ] Text readable at small screen (test on phone)
- [ ] Font consistent across all 6 images
- [ ] Each screenshot highlights 1 feature (not crowded)
- [ ] Real app UI visible (authentic)
- [ ] No personal data visible (blur if needed)
- [ ] Consistent color scheme (indigo/purple/gold)
- [ ] All screenshots at 1440×2560px minimum
- [ ] File size < 5MB each
- [ ] No watermarks/logos (except app logo)
- [ ] Grammar checked (no typos)

---

## Common Mistakes to Avoid

❌ **Small text** → Unreadable on Play Store preview
❌ **Too much text** → Users won't read it
❌ **Mockups** → Feels fake, use real screenshots
❌ **Inconsistent fonts** → Looks unprofessional
❌ **Personal data** → Privacy risk (blur/hide emails)
❌ **No CTA** → Users don't know what to do
❌ **Dark text on dark** → Zero contrast
❌ **All features at once** → Overwhelming, not focused
